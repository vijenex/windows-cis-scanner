## üêõ Issue Reporting

We welcome and encourage issue reports! Help us improve the tool by reporting bugs and suggesting features.

### Before Reporting

1. **Search existing issues** to avoid duplicates
2. **Update to latest version** if possible
3. **Review documentation** and README first

### Bug Reports

When reporting bugs, please include:

#### System Information
```
- Windows Version: (e.g., Windows Server 2025 Datacenter)
- Build Number: (e.g., 10.0.26100)
- PowerShell Version: $PSVersionTable.PSVersion
- Tool Version: (from README or release)
```

#### Error Details
```
- Full error message (copy/paste)
- Steps to reproduce
- Expected behavior
- Actual behavior
- Command line used
- Milestone files involved
```

#### Logs and Output
- Attach relevant log files
- Include PowerShell error output
- Sanitize any sensitive information

### Feature Requests

For new features, please describe:

- **Use case**: What problem does this solve?
- **Business value**: Why is this important?
- **Proposed solution**: How should it work?
- **Alternatives considered**: Other approaches you've thought of

### Security Issues

For security vulnerabilities:

1. **DO NOT** create public issues
2. **Contact repository maintainers** privately
3. **Provide detailed information** about the vulnerability
4. **Allow time** for assessment and fix before disclosure

## üîÑ Issue Lifecycle

1. **Submitted** - Issue created by user
2. **Triaged** - Maintainers review and label
3. **In Progress** - Work begins on resolution
4. **Testing** - Fix is being validated
5. **Resolved** - Issue is closed with solution

## üìã Issue Labels

- `bug` - Something isn't working
- `enhancement` - New feature or request
- `documentation` - Improvements to docs
- `question` - Further information requested
- `duplicate` - This issue already exists
- `wontfix` - This will not be worked on
- `priority-high` - Critical issues
- `priority-medium` - Important issues
- `priority-low` - Nice to have

## ‚úÖ Quality Guidelines

### Good Issues
- Clear, descriptive titles
- Detailed descriptions
- Steps to reproduce (for bugs)
- System information included
- Relevant logs/screenshots
- Respectful tone

### Issues We May Close
- Duplicate reports
- Insufficient information
- Off-topic discussions
- Requests for code modifications
- License violations
- Spam or inappropriate content
